# 設定画面（モーダル）実装ガイド

## 概要
Material UIを使用したモダンな設定画面をモーダル形式で実装しました。アプリケーションの右上に歯車アイコンがあり、これをクリックすることで設定画面が表示されます。

## 実装内容

### 1. 主なコンポーネント

#### Setting.tsx
`src/BIGsign/Setting.tsx` - 設定モーダルのメインコンポーネント

**機能:**
- 左側のサイドバーに6つのセクションがある
- 各セクションをクリックすると、右側の内容が切り替わる
- Material UIのDialogコンポーネントを使用してモーダル表示

**セクション一覧:**
1. **プロフィール** - 表示名、メール、自己紹介の設定
2. **ゲーム設定** - 自動保存、FPS表示、アニメーション速度、難易度
3. **サウンド設定** - マスターボリューム、BGM、SE、ボイスボリュームの調整
4. **グラフィックス** - 解像度、品質、フレームレート、VSync、パーティクル効果
5. **プライバシー** - プロフィール公開、フレンドリクエスト、オンラインステータス、メッセージ受け取り
6. **情報** - アプリケーション情報とリンク

### 2. App.tsxの修正

```tsx
// インポート追加
import { Setting } from './BIGsign/Setting'
import { IconButton, Box } from '@mui/material'
import { Settings as SettingsIcon } from '@mui/icons-material'

// ステート追加
const [openSettings, setOpenSettings] = useState(false);

// UIに歯車アイコンを追加
<IconButton
  onClick={() => setOpenSettings(true)}
  sx={{ mr: 2, color: 'inherit' }}
  title="設定"
>
  <SettingsIcon />
</IconButton>

// モーダルコンポーネント
<Setting open={openSettings} onClose={() => setOpenSettings(false)} />
```

### 3. 使用されたコンポーネント

- **Dialog** - モーダル表示
- **Box** - レイアウト
- **List/ListItem** - メニューリスト
- **TextField** - テキスト入力
- **Slider** - スライダーコントロール
- **Switch** - トグルスイッチ
- **Button** - ボタン
- **FormControlLabel** - ラベル付きフォーム要素
- **Paper** - コンテナ

### 4. スタイリング

**モーダルの特徴:**
- 背景が暗くなる（Backdrop）
- モーダル外の領域をクリックすると閉じる
- Escキーで閉じられる
- レスポンシブデザイン（モバイル対応）

**サイドバーの特徴:**
- 選択されたセクションは青い背景と左ボーダーでハイライト
- ホバー時に背景色が変わる
- アイコン付きのメニュー項目

## 設定の変更方法

現在の実装では、各設定セクションに「変更を保存」ボタンがあります。今後の実装において、以下の方法が考えられます：

### 推奨される変更方法

1. **リアルタイム更新**
   - スイッチやスライダーを変動させると即座に反映
   - バックエンドに自動保存

2. **保存ボタン方式** (現在)
   - 変更を確認してから保存
   - キャンセル機能の追加も可能

3. **ローカルストレージ連携**
   - ブラウザのローカルストレージに設定を保存
   - オフライン時も設定が保持される

4. **API連携**
   - バックエンドAPIにPOSTリクエストで保存
   - データベースに永続化

## トラブルシューティング

### モーダルが表示されない場合
- App.tsxで `openSettings` ステートが正しく管理されているか確認
- `Setting` コンポーネントが正しくインポートされているか確認

### スタイルが反映されない場合
- Material UIが正しくインストールされているか確認
- `@emotion/react` と `@emotion/styled` のバージョンが一致しているか確認

### CORSエラーが表示される場合
- バックエンドのCORS設定を確認
- ブラウザのコンソールでエラーメッセージを確認
